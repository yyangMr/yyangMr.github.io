使用函数处理数据

大多数 sql 支持一下类型的函数：

  • 用于处理文本字符串(如删除或填充值，转换值为大写或小写)的文本函数
  • 用于在数值数据上进行算数操作（如返回绝对值，进行代数运算）的数值函数
  • 用于处理日期和时间值并从这些值中提取特定成分的日期和时间函数

常用的文本处理函数如 LEFT() LENGTH() LOWER() LTRIM() RIGHT():返回字符串右边的字符 UPPER() RTRIM() SOUNDEX()

聚集函数 AVG() COUNT() MAX() MIN() SUM()

使用 count(*) 对表中行的数目进行行计数，不管表列中包含的是空值还是非空值,同 count(1)
使用 count(列名) 对特定列中具有值的行进行行计数，忽略NULL值

MAX()一般用来找出最大的数值或日期，但许多 DBMS 允许将它用来返回任意列中的最大值，包括返回文本列中的最大值，在用于文本数据时，MAX() 返回按该列排序后的最后一行，MIN()同理

sql数据类型

字符串数据类型

定长字符串接受固定长度的字符串，其长度是在创建表时指定的。
变长字符串存储任意长度的文本。
既然变长数据类型这样灵活，为什么还要用定长数据类型？答案是性能，DMMS 处理定长列远比处理变长列快得多，此外，许多DBMS不允许对变长列进行索引，这也会极大的影响性能。

数据类型 说明
CHAR 1~255个字符的定长字符串，长度必须在创建时规定
NCHAR CHAR的特殊形式，用来支持多字节或是Unicode字符
NVARCHAR TEXT的特殊形式，用来支持多字节或Unicode字符
TEXT 变长文本

数值数据类型

数据类型 说明
BIT 单个二进制位，或者为0或者为1，主要用于开关标志
DECIMAL(或NUMERIC) 定点或精度可变的浮点型
FLOAT(NUMBER) 浮点型
INT(或INTEGER) 四字节整数值
REAL 四字节浮点型
SMALLINT 二字节整数值
TINYINT 一字节整数值

多数DBMS支持一种用来存储货币值的特殊数值数据类型，一般记为MONEY 或者 CURRENCY。这些数据类型基本上时有特定取值范围的DECIMAL数据类型

日期和时间数据类型

数据类型 说明
DATE 日期值
DATETIME(或TIMESTAMP) 日期时间值
SMALLDATETIME 日期时间值，精确到分，无秒或毫秒
TIME 时间值

二进制数据类型

数据类型 说明
BINARY 定长二进制数据
LONG RAW 变长二进制数据，最长2GB
